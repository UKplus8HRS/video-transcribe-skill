Local ASR for Codex

Turn video links into transcripts Codex can actually read.

A small, deterministic skill for Bilibili, YouTube, generic URLs, and local media. It extracts audio, normalizes it with FFmpeg, runs local faster-whisper, and gives Codex clean text to summarize.

Bilibili native path

BV ids are resolved through Bilibili public web APIs. The script downloads DASH audio directly and avoids fragile Bilibili yt-dlp behavior.

Generic fallback

YouTube and other supported platforms are delegated to yt-dlp, keeping extraction broad without reimplementing every platform.

Local transcription

faster-whisper runs on CPU with int8 compute by default, so the workflow does not require a cloud API key.

Install

Drop the skill into Codex.

Clone the repo, install FFmpeg, create a small local Whisper environment, then invoke $video-transcribe whenever a video link needs to become text.

git clone https://github.com/UKplus8HRS/video-transcribe-skill.git
winget install --id Gyan.FFmpeg --exact
pip install -r requirements.txt

中文介绍

把视频链接变成 Codex 能读的文字。

这个 skill 先用确定性代码完成平台解析、音频下载、FFmpeg 转码和本地 Whisper 转写， 然后再让模型基于转写文本做摘要、提取和问答。B 站走公开 API 获取音频流，YouTube 和通用链接走 yt-dlp，本地文件则直接转码。

不依赖云端 key

默认使用本地 faster-whisper，适合临时视频摘要和私有素材处理。

适合无字幕视频

当 B 站页面显示“暂无字幕”时，仍然可以从音频生成可读转写。

输出可追溯

每行转写都有时间戳，并保存 metadata，方便回看和核对。
